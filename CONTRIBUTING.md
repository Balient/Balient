# Contributing to Balient

1. Keep commits focused.
2. Build before opening a pull request.
3. Do not commit APKs, keystores, SDK tools, or downloaded Bale APKs.
4. Document every smali hook with the class/method and reason for the patch.
5. When porting to a newer Bale version, update `docs/PORTING_CHECKLIST.md` with any changed anchors.

Recommended PR sections:

- Base Bale version.
- Features changed.
- Files touched.
- Build/sign result.
- Device smoke-test result.
- Known issues.
